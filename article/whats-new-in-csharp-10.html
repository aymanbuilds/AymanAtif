What’s New in C# 10: Features and Improvements

Explore the latest enhancements in C# 10, including global using directives, file-scoped namespaces, record improvements, and other modern features that simplify coding and improve maintainability.

Introduction

C# 10, released as part of .NET 6, introduces several powerful features and enhancements aimed at simplifying code, improving readability, and enhancing maintainability. These updates make C# even more concise and developer-friendly, allowing you to write clean, modern applications with less boilerplate.

In this article, we’ll explore the most important features in C# 10, including global using directives, file-scoped namespaces, improvements to records, constant interpolated strings, and more. We’ll also provide practical code examples to help you apply these features in real projects.

Global Using Directives

One of the most notable features in C# 10 is global using directives. They allow you to declare namespaces that apply across the entire project, reducing repetitive using statements in every file. 


// Before C# 10
using System;
using System.Collections.Generic;

namespace MyApp
{
    class Program
    {
        static void Main()
        {
            Console.WriteLine("Hello World!");
        }
    }
}

// With C# 10 - Global Usings
global using System;
global using System.Collections.Generic;

namespace MyApp
{
    class Program
    {
        static void Main()
        {
            Console.WriteLine("Hello World!");
        }
    }
}

By moving common namespaces to a single global file (e.g., GlobalUsings.cs), you make your codebase cleaner and reduce clutter in each class file.

File-Scoped Namespaces

C# 10 introduces a shorter syntax for namespaces, allowing all code in a file to belong to a single namespace without additional indentation. 


// Traditional syntax
namespace MyApp
{
    class Program
    {
        static void Main() { }
    }
}

// File-scoped namespace in C# 10
namespace MyApp;

class Program
{
    static void Main() { }
}

This small change improves readability, especially in files with multiple classes or long implementations.

Record Improvements

Records in C# are used for immutable data objects. C# 10 adds enhancements like with-expressions for structs and parameterless record structs, making it easier to use value types immutably. 


// Record struct in C# 10
public readonly record struct Point(int X, int Y);

var point1 = new Point(5, 10);
var point2 = point1 with { X = 20 };

Console.WriteLine(point2); // Output: Point { X = 20, Y = 10 }

These changes provide more flexibility and performance improvements when working with immutable data types.

Constant Interpolated Strings

In previous versions, string interpolation couldn’t be used in const declarations. C# 10 allows this, making compile-time string construction easier. 


// C# 10
const string Name = "Ayman";
const string Greeting = $"Hello, {Name}!";

Console.WriteLine(Greeting); // Output: Hello, Ayman!

This feature reduces boilerplate code when building fixed string messages and helps maintain constants more efficiently.

Lambda Expression Improvements

C# 10 enhances lambda expressions with natural types for lambdas and lambda return types. This makes them more versatile and easier to use in functional programming scenarios. 


// Lambda with natural return type
Func square = x => x * x;
Console.WriteLine(square(5)); // Output: 25

Extended Pattern Matching

Pattern matching becomes even more expressive in C# 10, allowing logical patterns, relational patterns, and improved switch expressions. 


int score = 85;

string grade = score switch
{
    >= 90 => "A",
    >= 80 => "B",
    >= 70 => "C",
    _ => "F"
};

Console.WriteLine(grade); // Output: B

These improvements reduce conditional complexity and make code more readable.

Conclusion

C# 10 brings several modern features that simplify code, reduce boilerplate, and improve developer productivity. Global using directives, file-scoped namespaces, enhanced records, and extended pattern matching are just a few examples that make coding in C# cleaner and more efficient.

By adopting these features, you can write more maintainable, concise, and expressive C# code — keeping your applications modern and aligned with the latest .NET practices.
